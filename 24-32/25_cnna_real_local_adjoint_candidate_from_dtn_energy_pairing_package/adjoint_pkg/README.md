# CNNA Test: real local adjoint candidate from DtN energy pairing

This package contains the next diagnostic after the longitudinal/transverse real-operator-calculus gate.

Script:

```bash
python3 test_real_local_adjoint_candidate_from_dtn_energy_pairing.py --max-level 6 --out real_local_adjoint_candidate_out_L6
```

Default run:

- true event-resolved ternary growth;
- sequential birth;
- parent-line + older-sibling environment at birth;
- immediate newborn backreaction on ancestors and older siblings;
- descendant shell-load aging;
- dynamic Schur/DtN record/live layer;
- double-history suffix quotient as transverse handoff control.

The test builds a candidate energy pairing on each local Env/UV role plane:

```text
G_p = B_p^T D_p B_p
```

and defines the formal candidate adjoint for `A : E_p -> E_q` by

```text
A* = G_p^{-1} A^T G_q.
```

The nontrivial gate is not the formula itself. The gate is whether the adjoints of the generated longitudinal and transverse operators are still visible inside the operator system generated by growth.

This test does not derive `J`, does not test `J^2=-I`, and does not claim a `C*`-algebra.
