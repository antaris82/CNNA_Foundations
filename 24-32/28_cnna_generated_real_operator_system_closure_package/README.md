# CNNA generated real operator-system closure diagnostic

This package contains the next gate after `record_live_block_adjoint_closure`.
It tests whether the **finite real operator system generated by actual growth-produced longitudinal and transverse Record/Live block operators** is stable under:

1. real linear span,
2. products up to finite degree,
3. the block DtN adjoint.

It deliberately does **not** test or set `J`, `i`, or `J^2 = -I`.

## Main script

```bash
python3 test_generated_real_operator_system_closure.py \
  --max-level 5 \
  --out generated_real_operator_system_out_L5 \
  --max-pairs-per-control 10 \
  --degrees 1 2 3 \
  --word-cap 30 \
  --mult-sample-cap 30
```

The script uses `record_live_block_base.py`, included in this package, for the growth model and Record/Live block operators.

## Primary result file

```text
generated_real_operator_system_out_L5/RESULTS_generated_real_operator_system_closure.md
```

## Interpretation

A positive pre-* gate would require small adjoint residuals for the generated word-space and small one-more-product residuals. The current L5 sample is not positive: finite products of the current L/T block generators still do not yield a robust growth-visible real *-closed operator system.
